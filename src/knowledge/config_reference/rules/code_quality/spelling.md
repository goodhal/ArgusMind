# 拼写错误检测规则

## 检测类型

### 1. 标识符拼写错误
- **变量名**
- **函数/方法名**
- **类名**
- **常量名**

### 2. 字符串内容拼写错误
- **日志消息**
- **错误消息**
- **用户可见文本**

## 检测原则

### 不报告的情况
- **引用位置**：只在声明位置报告，不在引用位置报告
- **命名约定**：符合项目命名约定的名称不报告
- **专有名词**：已知的技术术语、品牌名等

### 报告条件
- 明显的拼写错误（如 `usre` 应为 `user`）
- 影响可读性的拼写错误
- 可能导致理解错误的拼写错误

## 严重级别判定

| 类型 | 严重级别 | 说明 |
|------|----------|------|
| 标识符拼写错误 | low | 影响代码可读性 |
| 日志/错误消息拼写错误 | low | 影响问题排查 |
| 用户可见文本拼写错误 | medium | 影响用户体验 |

## 常见错误模式

- `recieve` → `receive`
- `accomodate` → `accommodate`
- `occurence` → `occurrence`
- `priviledge` → `privilege`
- `seperate` → `separate`
- `enviroment` → `environment`
- `definately` → `definitely`
- `existance` → `existence`
