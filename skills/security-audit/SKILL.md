# 安全审计

## 任务
对代码库进行全面的安全审计，识别潜在的漏洞和安全风险。

## Decision Tree

使用此决策树路由到正确的参考文件：

- **SQL 注入** → `references/sql-injection.md`
- **命令注入** → `references/command-injection.md`
- **XSS** → `references/xss.md`
- **路径遍历** → `references/path-traversal.md`
- **SSRF** → `references/ssrf.md`
- **XXE** → `references/xxe.md`
- **不安全反序列化** → `references/insecure-deserialization.md`
- **弱加密** → `references/weak-crypto.md`
- **认证/授权问题** → `references/auth-issues.md`
- **通用审计流程** → `references/general-process.md`

## 流程

1. **项目概览**：
   - 使用 `list_files` 了解项目结构
   - 识别技术栈和框架

2. **快速扫描**：
   - 使用 `ripgrep_search` 搜索常见危险模式
   - 搜索关键词：`eval`, `exec`, `system`, `popen`, `subprocess`, `pickle`, `yaml.load` 等
   - 搜索 SQL 查询、用户输入处理、认证相关代码

3. **漏洞类型检查**：
   - **SQL 注入**：查找未参数化的 SQL 查询
   - **命令注入**：查找使用用户输入执行命令的代码
   - **XSS**：查找未转义的用户输入输出
   - **路径遍历**：查找使用用户输入构造文件路径的代码
   - **SSRF**：查找向用户提供的 URL 发起请求的代码
   - **XXE**：查找 XML 解析代码
   - **不安全反序列化**：查找 pickle、jsonpickle、YAML 反序列化
   - **弱加密**：查找使用弱加密算法的代码
   - **认证/授权问题**：检查认证和授权逻辑

4. **深度分析**：
   - 使用 `code_search` 查找关键类和方法
   - 使用 `read_file` 读取关键文件
   - 分析数据流：用户输入 → 处理 → 输出/存储

5. **生成报告**：
   - 按严重性列出发现
   - 提供修复建议
   - 包含代码片段和位置

## Anti-Patterns

避免以下常见错误：

### ❌ 猜测式分析
- **错误**：假设某个函数存在漏洞而不验证
- **正确**：使用工具读取代码，确认漏洞存在

### ❌ 忽略上下文
- **错误**：只看单行代码，不考虑整体上下文
- **正确**：分析完整的数据流和调用链

### ❌ 过度报告
- **错误**：报告所有潜在风险点，不区分严重性
- **正确**：按严重性和可利用性分类报告

### ❌ 缺少证据
- **错误**：只说"存在漏洞"，不提供代码证据
- **正确**：包含具体的代码片段和位置

### ❌ 忽略防护措施
- **错误**：发现危险函数就报告漏洞
- **正确**：检查是否有有效的防护措施（sanitizer、validator 等）

### ❌ 不验证可达性
- **错误**：报告不可达的代码漏洞
- **正确**：确认漏洞代码可被外部触发
